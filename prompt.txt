Add Confirmation Before Final Apply
Currently actions happen immediately.
Better approach:
New Block: REVIEW
Before applying changes:
Please review changes:
IAM User: bhavya-dev
S3 Bucket: aktive-test-s3
Policy: s3-access-policy
Attach Policy: YES
Proceed?
1. Yes
2. No (Rollback & Exit)
This avoids accidental creation

Add Logging
Very useful.
Add log file:
LOG_FILE="iam-automation.log"
Example:
echo "[INFO] Creating IAM User..." | tee -a $LOG_FILE
This helps debugging

mentioned checking names.
Good.
But add regex validation too.
IAM Username Validation
AWS IAM usernames allow:
[a-zA-Z0-9+=,.@_-]
S3 Bucket Naming Rules
Must be:
globally unique
lowercase
3–63 chars
no uppercase
no underscores
Policy Name Validation
Avoid spaces.
6. Add Region Selection
For S3 + CodeCommit.
Because CodeCommit ARN depends on region.
Add:
Select AWS Region:
1. ap-south-1
2. us-east-1
3. custom
Store:
AWS_REGION="ap-south-1"

Create:
Option 1 — ReadOnly
Option 2 — ReadWrite
Option 3 — FullBucketAdmin
Much safer.
Because currently your policy allows:
bucket policy modification
ACL changes
replication
access point creation
public access block changes
Developer users should not get these usually.
8. Use Policy Templates Dynamically
Instead of hardcoding bucket/repo names in JSON.
Better:
Create template JSON.
Example:
"Resource": [
  "arn:aws:s3:::${BUCKET_NAME}",
  "arn:aws:s3:::${BUCKET_NAME}/*"
]
Then replace dynamically.
Same for CodeCommit.
